HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • HTML stands for Hyper Text Markup Language
  • HTML elements tell the browser how to display the content
  • An HTML element is defined by a start tag, some content, and an end tag.
  • Some HTML elements have no content eg: line break (br). They are called empty elements and don't have an end tag.
  • Some HTML elements used to add content to a webpage include text, header, image, and list elements.
  • Other HTML element types used to divide and describe a page include meta-related elements, content division elements, and semantic elements.
  • An HTML attribute defines additional information about an element (ex:lang="en"), they are always defined in the beginning element, never the closing one.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Class attributes allow us to share a CSS rule to any element we choose by assigning the rule to a class attribute with a class selector.
  • An inline CSS is used when a developer wants to style an element directly inthe HTML file.
  • Inline styles apply to specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • The internal style sheet allows you to include CSS code in "head" section of a markup document.
  • External stylesheet works by linking a .css file, containing all the CSS rules, to an HTML document.
  • The CSS box model demonstrates that every HTML element is a box. Each box consists of margins outside the element, and the element's borders, padding and content.
  • Content inside of padding, padding inside of border, and border inside of margin.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch name: creates a new branch and switches to it
  • git add [file]: add a file as it looks now to your next commit
  • git commit -m "[descriptive message]": commit your staged contentas a new commit snapshot
  • git branch: list your branches. a * will appear next to the currently active branch
  • git branch [branch name]: create a new branch at the current commit
  • git checkout: switch to another branch and check it out into your working directory
  • git merge [branch]: merge the specified branch's history into the current one
  • git log: show all commits in the current branch's history

Javascript

  • Javascript (JS) is a high-level dynamic programming language that allows users to interact with websites.
  • Using Javascript, you can power websites with effects, interactivity, data communication and more.
  • Null is a special value in JS that refers to the absence of a value.
  • Primitive values: immutable datum represented directly at the lowest level of the language. All types except objects define immutable values (values that can't be changed, e.g. Strings are immutable, once created they cannot be modified)
  • Boolean represents a logical entity and can have two values: true and false.
  • Undefined type is a variable that has not been assigned a value.
  • Numeric types include Number, BigInt and NaN.
  • JS's String type is used to represent textual data. It is a set od "elements" of 16-bit-unsigned integer values. Each element in the String occupies a position in the String. (e.g. first element is at index 0, next at index 1)
  • The length of a String is the number of elements in it.
  • It is possible to create another string based on an operation on the original string. A substring of the original by picking indiviudal letters or using substring() or a concentration of two strings using the concentration operator (+) or concat().
  • Symbol type is a unique and immutable primitive value and may be used as the key of an Object property. In some programming languages, Symbols are called "atoms".
  • An object is a value in memory which is possibly referenced by an identifier.
  • An identifier is a sequence of characters in the code that identifies a variable, function or property. In JS, identifiers are case-sensitive and contain Unicode letters,$,_, and digits (0-9), but may not start with a digit.
  • An identifier differs froma string inthat a string is data, while an identifier is part of the code.
  • In JS, there is no way to convert identifiers to strings, but sometimes it is possible to parse strings into identifiers.
  • In JS, objects can be seen as a collection of properties. Property values can be values of any type, including other objects.
  • Properties are identified using key values. A key value is either a string value (a sequence of characters used to represent text) or a symbol value (a built-in object whose constructor retunrs a primitive thats guaranteed to be unique).
  • Symbols are often used to add unique property keys to an object that won't collide with keys any other code might add to the object.
  • In JS, a primitive (primitive value, primitive data type) is data that is not an object and has no methods of properties. There are 7 primitive data types: string, number, bigint, boolean, undefined, symbol and null.
